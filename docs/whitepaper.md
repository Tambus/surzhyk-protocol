# Суржик — Whitepaper-конспект

**Версія:** 0.1 (драфт для подачі на Web3 Resilience Lab Ukraine, травень 2026)
**Засновник:** Bohdan / @Tambus
**Гасло:** *Це не російська, це український суржик.*

---

## 1. Проблема

Два століття суржик нав'язувався як «помилка» та «безкультурне мовлення», що породило:

- **Мовну дискримінацію** всередині українського суспільства
- **Регіональне розділення** за лінгвістичним стигматом
- **Беззахисність живої народної мови** перед російською асиміляцією
- **Відсутність економічного стимулу** для носіїв зберігати регіональну лексику

Понад половина населення України говорить суржиком щодня. Цей мовний пласт не має ні системного збереження, ні інструменту легітимізації, ні економічної моделі для його носіїв.

## 2. Рішення

**«Суржик»** — соціальний протокол на блокчейні, що:

1. Переосмислює суржик як **саморізновид мови** (як галицький, полтавський чи буковинський діалект), а не «помилку».
2. Карбує кожне суржикове слово як **NFT** з обов'язковим зазначенням значення, регіону, автора.
3. Створює **геометрично-спадний потік роялті** оригінальному карбувачу та послідовним контрибуторам.
4. Будує **soulbound літопис** автора — багаторівневу репутацію, побудовану на унікальності та якості внесків у мовну спадщину.
5. Закладає фундамент під майбутній шар **громадянського голосу** з регіональною верифікацією через «Дію».

## 3. Архітектура токенів

### 3.1. `$SURZH` — utility token (ERC-20)

Єдиний обіговий токен протоколу. Використовується для:

- Карбу нового слова (~10 $SURZH)
- Query — спроби карбу існуючого слова (1 $SURZH)
- Стейку на пропозицію доповнення (5 $SURZH)
- Голосування у форум-тредах
- Газ-комісій усередині протоколу

**Дефляційний механізм:** 10% з кожного query спалюється.

**Faucet onboardingу:** новий користувач отримує стартовий запас `$SURZH` за проходження кроків:
- Реєстрація + KYC
- Self-declared регіон (базовий рівень)
- *(Roadmap)* «Дія»-verified регіон — повний faucet + permission карбувати regional слова з вищою вагою

### 3.2. NFT слова (ERC-721)

Кожне карбоване слово = унікальний NFT. Метадані on-chain + IPFS:
- слово, значення, регіон, дата карбу, оригінатор
- список схвалених контрибуторів зі стеком позицій
- зв'язки з кластером варіацій (якщо є)

### 3.3. Soulbound літопис (ERC-5114)

Non-transferable профіль користувача. Накопичує:
- унікальні карби (вага найвища)
- схвалені доповнення (вага середня)
- голоси у валідаціях (вага мала)
- штрафи за доведені фейки / помилки

## 4. Економіка карбу та доповнень

### 4.1. Карб нового слова

Автор сплачує `~10 $SURZH`. Запис on-chain + метадані на IPFS. Слово стає NFT з оригінатором #0.

### 4.2. Query existing word

Користувач намагається додати слово, яке вже існує:

| Отримувач | Частка |
|---|---|
| Оригінатор слова | 50% |
| DAO treasury | 40% |
| Burn (дефляція) | 10% |

### 4.3. Доповнення (propose-and-approve)

Контрибутор стейкає `5 $SURZH` на пропозицію доповнення (нове значення, етимологія, регіональна варіація, аудіо вимови).

- **Apprve:** доповнення активується, контрибутор стає частиною стеку роялті.
- **Reject:** застейкані `$SURZH` спалюються або переходять у DAO pool.

### 4.4. Геометрично-спадний стек роялті

Усі майбутні query та доповнення на слові розподіляються:

| Позиція в стеку | Частка |
|---|---|
| Оригінатор (#0) | 50% |
| Контрибутор #1 | 20% |
| Контрибутор #2 | 10% |
| Контрибутор #3 | 5% |
| Контрибутор #4 | 2.5% |
| Контрибутор #5+ | спадає геометрично |
| Залишок | DAO / burn |

### 4.5. Кластеризація варіацій

Фонетична близькість + семантичний збіг → пропозиція merge. Якщо apprve спільнотою — оригінатори всіх варіацій стають **co-originators кластера**, частки пропорційні date-of-mint.

UI: hover на значення підсвічує всі регіони, де воно вживається у відповідній формі.

## 5. Валідація та антифрод

### 5.1. Soft path — голосування спільноти

Швидкий шлях для простих випадків: +/– голоси користувачів зі схваленими літописами.

### 5.2. Hard path — Підтверджувачі

Експерти з підтвердженою репутацією: філологи, краєзнавці, журналісти. Їхній голос важить значно більше. Bootstrap — особиста мережа + партнерство з філологічними факультетами КНУ ім. Шевченка, ЛНУ ім. Франка, Острозької академії.

**Anti-corruption baked in:** Підтверджувач може втратити статус, якщо його рішення системно не витримують перевірки post-factum.

### 5.3. Dispute forum

Кожне слово та доповнення має тред. Аргументи on-chain. Невирішені спори ескалюються до голосування або Підтверджувача.

### 5.4. Slashing

Якщо доповнення доведено помилковим:
- Refund сплачених комісій попереднім контрибуторам
- Заморозка майбутніх комісій по цій позиції
- Мінус літопис автору пропозиції

## 6. Технологічний стек

- **Smart contracts:** Solidity на BNB Smart Chain; критичні операції — opBNB L2
- **Контракти:** ERC-721 (NFT), ERC-20 (`$SURZH`), кастомний роялті-розподільник, soulbound літопис (ERC-5114)
- **Метадані:** IPFS (Pinata)
- **Індексація:** The Graph, PostgreSQL
- **Фронт:** Next.js, TailwindCSS, wagmi/ethers.js, Web3Auth
- **AI-шар:** sentence embeddings для фонетичної кластеризації (multilingual BERT)
- **Roadmap-інтеграції:** «Дія.Підпис», BlockPass
