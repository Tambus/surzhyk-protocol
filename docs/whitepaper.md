# Суржик — Whitepaper-конспект

**Версія:** 0.1 (драфт для подачі на Web3 Resilience Lab Ukraine, травень 2026)
**Засновник:** Bohdan / @Tambus
**Гасло:** *Це не російська, це український суржик.*

---

## 1. Проблема

Два століття суржик нав'язувався як «помилка» та «безкультурне мовлення», що породило:

- **Мовну дискримінацію** всередині українського суспільства
- **Регіональне розділення** за лінгвістичним стигматом
- **Беззахисність живої народної мови** перед російською асиміляцією
- **Відсутність економічного стимулу** для носіїв зберігати регіональну лексику

Понад половина населення України говорить суржиком щодня. Цей мовний пласт не має ні системного збереження, ні інструменту легітимізації, ні економічної моделі для його носіїв.

## 2. Рішення

**«Суржик»** — соціальний протокол на блокчейні, що:

1. Переосмислює суржик як **саморізновид мови** (як галицький, полтавський чи буковинський діалект), а не «помилку».
2. Карбує кожне суржикове слово як **NFT** з обов'язковим зазначенням значення, регіону, автора.
3. Створює **геометрично-спадний потік роялті** оригінальному карбувачу та послідовним контрибуторам.
4. Будує **soulbound літопис** автора — багаторівневу репутацію, побудовану на унікальності та якості внесків у мовну спадщину.
5. Закладає фундамент під майбутній шар **громадянського голосу** з регіональною верифікацією через «Дію».

## 3. Архітектура токенів

### 3.1. `$SURZH` — utility token (ERC-20)

Єдиний обіговий токен протоколу. Використовується для:

- Карбу слова-вузла (центрального значення кластера)
- Стейку на пропозицію доповнення (додавання регіональної форми у кластер)
- Голосування у форум-тредах
- Газ-комісій усередині протоколу

**Дефляційний механізм:** частина комісій з кожного доповнення спалюється (баланс burn-частки калібрується на основі обсягу активності протоколу).

**Faucet onboardingу:** новий користувач отримує стартовий запас `$SURZH` за проходження кроків:
- Реєстрація + KYC
- Self-declared регіон (базовий рівень)
- *(Roadmap)* «Дія»-verified регіон — повний faucet + permission карбувати regional слова з вищою вагою

### 3.2. NFT кластера слова (ERC-721)

Кожен кластер значення = унікальний NFT. Метадані on-chain + IPFS:
- центральне значення
- літературна укр. форма (або `null`, якщо нормативного еквіваленту немає — тоді першокарбована регіональна форма стає центром)
- куратор (#0) + список схвалених контрибуторів з регіональними формами
- лічильник переглядів, зважений по літопису переглядаючих
- зв'язки зі спорідненими кластерами (наприклад, синонімічні або похідні)

### 3.3. Soulbound літопис (ERC-5114)

Non-transferable профіль користувача. Накопичує:
- куратурство кластерів (вага найвища)
- схвалені доповнення регіональних форм (вага середня)
- голоси у валідаціях (вага мала)
- штрафи за доведені фейки / помилки

## 4. Економіка кластера та доповнень

### 4.1. Перегляд — безкоштовний

Browse словника вільний для всіх. Кожен перегляд кластера інкрементує `view_counter`. Лічильник **зважений по літопису** того, хто переглядає: анонімний користувач — мінімальний вклад, користувач з високим літописом — більший. Це дає чесний рейтинг популярності, стійкий до накрутки ботами.

### 4.2. Карб слова-вузла

Творець кластера сплачує `~10 $SURZH`. On-chain записується:
- центральне значення
- літературна укр. форма (за замовчуванням — нормативна форма зі словника еталонів; для слів без еквіваленту першокарбована регіональна форма стає центром)
- регіон творця, дата, автор

Творець стає **куратором кластера** (позиція #0 у стеку роялті).

### 4.3. Додавання регіональної форми (propose-and-approve)

Контрибутор стейкає `~5 $SURZH` на пропозицію додати регіональну форму у існуючий кластер. Чекає схвалення:

- **Soft path:** голосування спільноти (швидко, прості випадки)
- **Hard path:** Підтверджувач (експерт-філолог / краєзнавець / журналіст з підтвердженою репутацією) — для спірних

Якщо apprve — контрибутор стає частиною стеку роялті кластера. Якщо reject — застейкані `$SURZH` спалюються або переходять у DAO pool.

### 4.4. Геометрично-спадний стек роялті на доповненнях

Коли новий контрибутор додає чергову регіональну форму у кластер — його комісія розподіляється вгору по стеку:

| Позиція в стеку | Частка |
|---|---|
| Куратор кластера (#0) | 50% |
| Контрибутор #1 | 20% |
| Контрибутор #2 | 10% |
| Контрибутор #3 | 5% |
| Контрибутор #4 | 2.5% |
| Контрибутор #5+ | спадає геометрично |
| Залишок | DAO / burn |

Це стимулює:
- **Першість** — творці кластерів отримують найбільшу частку
- **Раннє доповнення** — перші контрибутори формують довгий хвіст роялті
- **Якість центрального значення** — кластери з точним значенням притягують більше доповнень

### 4.5. Race condition і конфлікти кластерів

**Паралельні спроби карбу:** перемагає перший on-chain commit. Програла спроба отримує refund стейку мінус gas.

**Дублюючі кластери:** AI-шар фонетичної та семантичної кластеризації виявляє дублікати post-factum і пропонує merge. Якщо merge схвалено — куратори обох кластерів стають co-originators об'єднаного, частки пропорційні date-of-mint. Регіональні форми зливаються в єдину мапу.

### 4.6. Slashing помилок

Якщо доповнення доведено фейком чи помилкою через форум-тред:
- Refund сплачених контрибутором коштів попереднім учасникам стеку
- Заморозка майбутніх роялті позиції до резолюції
- Мінус літопис автору пропозиції
- При системних порушеннях — тимчасова блокіровка можливості карбувати

## 5. Валідація та антифрод

### 5.1. Soft path — голосування спільноти

Швидкий шлях для простих випадків: +/– голоси користувачів зі схваленими літописами.

### 5.2. Hard path — Підтверджувачі

Експерти з підтвердженою репутацією: філологи, краєзнавці, журналісти. Їхній голос важить значно більше. Bootstrap — особиста мережа + партнерство з філологічними факультетами КНУ ім. Шевченка, ЛНУ ім. Франка, Острозької академії.

**Anti-corruption baked in:** Підтверджувач може втратити статус, якщо його рішення системно не витримують перевірки post-factum.

### 5.3. Dispute forum

Кожне слово та доповнення має тред. Аргументи on-chain. Невирішені спори ескалюються до голосування або Підтверджувача.

### 5.4. Slashing

Якщо доповнення доведено помилковим:
- Refund сплачених комісій попереднім контрибуторам
- Заморозка майбутніх комісій по цій позиції
- Мінус літопис автору пропозиції

## 6. Технологічний стек

- **Smart contracts:** Solidity на BNB Smart Chain; критичні операції — opBNB L2
- **Контракти:** ERC-721 (NFT), ERC-20 (`$SURZH`), кастомний роялті-розподільник, soulbound літопис (ERC-5114)
- **Метадані:** IPFS (Pinata)
- **Індексація:** The Graph, PostgreSQL
- **Фронт:** Next.js, TailwindCSS, wagmi/ethers.js, Web3Auth
- **AI-шар:** sentence embeddings для фонетичної кластеризації (multilingual BERT)
- **Roadmap-інтеграції:** «Дія.Підпис», BlockPass

- **Культурна стійкість:** збереження живої регіональної мови як саморізновида
- **Інформаційна стійкість:** репутаційний фільтр проти ботів та інфо-вкидів
- **Інституційна стійкість:** прозорий on-chain літопис — фундамент майбутніх anti-corruption механізмів
- **Економічна стійкість:** новий клас цифрових активів на основі мовної спадщини
